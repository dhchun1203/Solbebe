# 관리자 페이지 설정 가이드

## 관리자 이메일 설정

관리자 페이지에 접근하려면 관리자 이메일을 설정해야 합니다.

### 방법 1: 환경 변수 설정 (권장)

프로젝트 루트의 `.env` 파일에 다음을 추가하세요:

```env
VITE_ADMIN_EMAILS=admin@solbebe.com,manager@solbebe.com
```

여러 관리자 이메일을 설정하려면 쉼표(,)로 구분하세요.

### 방법 2: 코드에서 직접 설정

`src/constants/index.js` 파일의 `ADMIN_EMAILS` 배열에 관리자 이메일을 추가하세요:

```javascript
export const ADMIN_EMAILS = [
  'admin@solbebe.com',
  'manager@solbebe.com',
]
```

## 관리자 기능

### 1. 관리자 로그인 제한

- 관리자 이메일로 등록된 계정만 로그인할 수 있습니다.
- 관리자가 아닌 이메일로 로그인을 시도하면 자동으로 로그아웃됩니다.

### 2. 관리자 대시보드 메뉴

- 관리자로 로그인하면 프로필 드롭다운 메뉴에 "관리자 대시보드" 메뉴가 표시됩니다.
- 클릭하면 관리자 대시보드로 이동합니다.

### 3. 관리자 페이지

#### 대시보드 (`/admin/dashboard`)
- 전체 상품 수
- 전체 문의 수
- 최근 7일 문의 수
- 빠른 액션 링크

#### 상품 관리 (`/admin/products`)
- 상품 목록 조회
- 상품 추가 (추후 구현)
- 상품 수정 (추후 구현)
- 상품 삭제

#### 문의 관리 (`/admin/inquiries`)
- 고객 문의 목록 조회
- 문의 상세 정보 확인

## 라우트 보호

관리자 페이지는 `AdminRoute` 컴포넌트로 보호됩니다:
- 로그인하지 않은 사용자: 홈으로 리다이렉트
- 관리자가 아닌 사용자: 홈으로 리다이렉트
- 관리자만 접근 가능

## 주의사항

1. **환경 변수 설정 후 재시작**: `.env` 파일을 수정한 후에는 개발 서버를 재시작해야 합니다.

2. **프로덕션 배포**: Vercel 등에 배포할 때는 환경 변수를 설정해야 합니다.

3. **보안**: 관리자 이메일은 민감한 정보이므로 `.env` 파일을 `.gitignore`에 추가하세요.

## Supabase 설정

관리자 계정을 Supabase에서 생성하려면:

1. Supabase 대시보드 → Authentication → Users
2. "Add user" 클릭
3. 관리자 이메일과 비밀번호 입력
4. 생성된 이메일을 `VITE_ADMIN_EMAILS`에 추가

## 문제 해결

### 관리자 메뉴가 보이지 않을 때

1. `.env` 파일에 `VITE_ADMIN_EMAILS`가 올바르게 설정되었는지 확인
2. 이메일 주소가 정확히 일치하는지 확인 (대소문자 구분 없음)
3. 개발 서버를 재시작했는지 확인
4. 브라우저 콘솔에서 에러 메시지 확인

### 로그인이 안 될 때

1. 관리자 이메일이 `VITE_ADMIN_EMAILS`에 포함되어 있는지 확인
2. Supabase에서 해당 이메일로 계정이 생성되어 있는지 확인
3. 이메일 인증이 완료되었는지 확인
