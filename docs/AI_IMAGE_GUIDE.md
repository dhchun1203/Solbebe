## 1. AI 이미지 생성 도구 선택

### 1-1. DALL·E 3 (OpenAI)

**장점**:
- 프롬프트 이해도 높음
- 일관된 스타일 생성 가능
- 빠른 생성 속도

**사용 방법**:
- [ChatGPT Plus](https://chat.openai.com) (유료) 또는 [Microsoft Image Creator](https://www.bing.com/images/create) (무료)

**프롬프트 구조**:
```
[주제] [스타일] [색상] [배경] [조명] [화질]
```

### 1-2. Midjourney

**장점**:
- 고품질 이미지
- 예술적 감성
- 다양한 스타일 옵션

**사용 방법**:
- [Midjourney Discord](https://midjourney.com) (유료)

**프롬프트 구조**:
```
/imagine prompt: [주제] [스타일] [색상] [배경] --ar 1:1 --style raw
```

### 1-3. Ideogram

**장점**:
- 텍스트 렌더링 우수
- 무료 사용 가능
- 빠른 생성

**사용 방법**:
- [Ideogram.ai](https://ideogram.ai) (무료/유료)

**프롬프트 구조**:
```
[주제] [스타일] [색상] [배경] [조명]
```

### 1-4. 추천

**초보자**: DALL·E 3 (Microsoft Image Creator) - 무료, 쉬움
**고품질**: Midjourney - 유료, 최고 품질

## 3. 이미지 스타일 통일 가이드

### 3-1. 일관된 스타일 유지 방법

#### 공통 요소

1. **배경**
   - 크림색 (#FFF8F0) 또는 베이지색 (#F5E6D3)
   - 깔끔하고 단순한 배경
   - 패턴이나 무늬 최소화

2. **조명**
   - 부드러운 자연광 느낌
   - 강한 그림자 없음
   - 전체적으로 밝고 따뜻한 톤

3. **구도**
   - Flat lay (평면 배치) 또는 스탠드 사용
   - 제품이 중심에 위치
   - 여백을 충분히 확보

4. **색상**
   - 파스텔 톤 유지
   - 과도한 채도 지양
   - 크림, 베이지, 핑크, 블루, 라벤더 등

### 3-2. 프롬프트 템플릿

**기본 템플릿**:
```
[제품명] in [색상] color, professional baby clothing product photography, 
soft pastel palette, displayed on [배경색] background, 
natural soft lighting, minimalist style, high quality commercial photo, 
fabric texture visible, warm cozy atmosphere
```

**예시**:
```
Baby bodysuit in soft pink color, professional baby clothing product photography, 
soft pastel palette, displayed on cream background, 
natural soft lighting, minimalist style, high quality commercial photo, 
organic cotton texture visible, warm cozy atmosphere
```

### 3-3. 각도별 프롬프트

#### 메인 이미지 (상품 카드용)
- Flat lay, 정면 뷰
- 제품 전체가 보이는 구도

#### 상세 페이지용 이미지
1. **정면 뷰**: "front view, full product visible"
2. **옆면 뷰**: "side view, showing depth and fit"
3. **디테일**: "close-up detail shot of fabric texture and stitching"
4. **착용 모습** (선택): "baby wearing the product, natural lifestyle photo"

## 4. 이미지 업로드 및 적용

### 4-1. 이미지 저장 방법

#### 방법 A: Supabase Storage 사용 (권장)

1. **Supabase Storage 버킷 생성**
   ```
   - Supabase 대시보드 → Storage
   - "New bucket" 클릭
   - 이름: "product-images"
   - Public bucket으로 설정
   ```

2. **이미지 업로드**
   - Storage → product-images → "Upload file"
   - 생성한 이미지 업로드
   - 업로드 후 URL 복사

3. **URL 형식**
   ```
   https://[project-id].supabase.co/storage/v1/object/public/product-images/[filename]
   ```

#### 방법 B: 외부 이미지 호스팅

**추천 서비스**:
- **Cloudinary** - 무료 플랜 제공, 이미지 최적화 기능
- **Imgur** - 무료, 간단함
- **ImageBB** - 무료 이미지 호스팅

#### 방법 C: 프로젝트 assets 폴더 (로컬)

```
src/assets/images/
  ├── products/
  │   ├── top/
  │   ├── bottom/
  │   ├── dress/
  │   └── accessory/
```

**단점**: 빌드 크기 증가, Vercel 배포 시 제한적

### 4-2. 데이터베이스에 이미지 URL 적용

#### 방법 1: Supabase Table Editor에서 직접 수정

1. Table Editor → products 테이블
2. 해당 상품의 `images` 필드 클릭
3. JSON 배열 형식으로 URL 입력:
   ```json
   ["https://example.com/image1.jpg", "https://example.com/image2.jpg"]
   ```

#### 방법 2: SQL로 일괄 업데이트

```sql
UPDATE products 
SET images = ARRAY[
  'https://your-image-url.com/product1-main.jpg',
  'https://your-image-url.com/product1-detail.jpg',
  'https://your-image-url.com/product1-side.jpg'
]
WHERE id = '상품-id-here';
```

## 5. 최적화 팁

### 5-1. 이미지 생성 시 주의사항

1. **해상도**
   - 최소: 800x800px
   - 권장: 1200x1200px
   - 최대: 2000x2000px (용량 고려)

2. **파일 형식**
   - JPG: 일반 제품 사진 (용량 작음)
   - PNG: 투명 배경 필요시

3. **파일명 규칙**
   ```
   [category]-[product-name]-[view]-[number].jpg
   예: top-bodysuit-pink-front-1.jpg
   ```

### 5-2. 이미지 편집 (필요시)

**무료 도구**:
- **Canva** - 간단한 편집, 크기 조정
- **Photopea** - Photoshop 무료 대안
- **Remove.bg** - 배경 제거

### 5-3. 프롬프트 개선 팁

1. **구체적인 색상 명시**
   - "soft pink" (비권장)
   - "soft rose pink, hex #FFE5E5" (권장)

2. **스타일 키워드 추가**
   - "commercial product photography"
   - "e-commerce style"
   - "minimalist"

3. **부정 프롬프트 사용** (일부 도구)
   - Midjourney: `--no shadows, harsh lighting, busy background`

### 5-4. 배치 생성 워크플로우

1. **템플릿 준비**
   - 카테고리별 기본 프롬프트 작성
   - 색상만 변경하며 반복 생성

2. **이미지 정리**
   - 생성된 이미지 폴더 구조화
   - 명명 규칙 적용

3. **품질 확인**
   - 각 이미지 검토
   - 스타일 일관성 확인
   - 재생성 필요한 항목 표시
