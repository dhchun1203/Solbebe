# 이미지 파일명 규칙 가이드

각 상품별 9장의 이미지(정면/사이드/디테일 × 3색상)에 대한 파일명 규칙입니다.

---

## 파일명 구조

```
[카테고리]-[상품명]-[색상]-[뷰].[확장자]
```

### 구성 요소

- **카테고리**: `top`, `bottom`, `dress`, `accessory`
- **상품명**: 영문 소문자, 하이픈으로 연결 (예: `bodysuit`, `long-sleeve-tshirt`)
- **색상**: `beige`, `pink`, `blue`
- **뷰**: `front`, `side`, `detail`
- **확장자**: `.png` (권장)

## 파일명 규칙 요약

1. **소문자 사용**: 모든 파일명은 소문자
2. **하이픈 구분**: 단어 간 하이픈(`-`) 사용, 공백 금지
3. **순서**: `카테고리-상품명-색상-뷰`
4. **일관성**: 모든 파일에 동일한 규칙 적용
5. **간결함**: 너무 길지 않게 (최대 50자 권장)

---

## 대안 파일명 규칙 (더 짧게)

더 간단한 파일명이 필요한 경우:

```
[카테고리]-[번호]-[색상]-[뷰].[확장자]
```

예시:
```
top-1-beige-front.png
top-1-pink-front.png
top-1-blue-front.png
top-1-beige-side.png
top-1-pink-side.png
top-1-blue-side.png
top-1-beige-detail.png
top-1-pink-detail.png
top-1-blue-detail.png
```

이 경우 데이터베이스나 별도 매핑 파일에 번호와 상품명 매칭 정보가 필요합니다.
